This is a **user research interview** run on Mom Test principles. The goal should be a learning outcome about the participant's real life and behavior ("understand how they currently handle returns and what it costs them") — never validation of the user's idea. Steer the checklist toward topics to mine about their existing behavior, workarounds, and pains — NOT a list of leading questions. Good labels: "Current returns workflow", "Last time it broke", "What they pay today". Remind the user the call is about the participant's life, not their idea; flag if a proposed goal is really "confirm they'll like my product".
